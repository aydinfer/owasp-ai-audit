General Controls

GEN-GOVERNANCE — governancecontrols

MEDIUM
Evidence
No mention of incident response process, model registry, or change-management for prompts or models in the architecture document.
Reasoning
Governance gaps. Not exploitable directly but compounds every other risk.
Recommended controls:
  • CTRL-AI-GOVERNANCE — AI governance: model registry, prompt versioning, incident response, change management.

Input Threats

INPUT-PROMPT-INJECTION — promptinjection

HIGH
Evidence
Architecture states 'user message concatenated into system prompt template'. No mention of input filtering, instruction hierarchy, or output validation.
Reasoning
Direct concatenation without delimiter discipline. No detection layer mentioned. No output validation. The retrieval step compounds risk because attacker-controlled content can also reach the prompt.
Recommended controls:
  • CTRL-PROMPT-INJECTION-LAYERS — Apply layered prompt injection defence — input filtering, instruction hierarchy, output validation, content provenance.

INPUT-INDIRECT-PROMPT-INJECTION — indirectpromptinjection

HIGH
Evidence
RAG retrieval pulls from a knowledge base updated by support agents. No mention of content sanitisation before inclusion in the LLM context.
Reasoning
Trusted-feeling sources (internal KB) are still injection vectors if any contributor is compromised or if any document contains untrusted content. The architecture treats KB content as trusted, which is the failure mode.
Recommended controls:
  • CTRL-PROMPT-INJECTION-LAYERS — Treat retrieved content as untrusted. Sanitise, mark provenance, isolate from instructions.

INPUT-SENSITIVE-DISCLOSURE — disclosureuseoutput

MEDIUM
Evidence
System prompt contains operational details about internal tooling. No output filter for these details.
Reasoning
Disclosable through prompt-leak attacks. Material risk but not catastrophic.
Recommended controls:
  • CTRL-OUTPUT-FILTERING — Output filtering and redaction layer for sensitive operational details.

Development-time

DEV-SUPPLY-CHAIN — supplychainmanage

MEDIUM
Evidence
Architecture references using an open-source embedding model from HuggingFace. No mention of signature verification, SBOM, or provenance tracking.
Reasoning
Third-party model with no integrity verification. Mitigation: model is widely-used and well-known, but no controls listed.
Recommended controls:
  • CTRL-SUPPLY-CHAIN — Model SBOM, signature verification, pinned versions, periodic re-validation.

Runtime

RUN-AUGMENTATION-MANIPULATION — augmentationmanipulation

HIGH
Evidence
Knowledge base is editable by support staff (per Section 2.4). No two-person review on KB changes. Embeddings re-indexed automatically on KB change.
Reasoning
A single insider or compromised account can poison the retrieval surface for the entire user base. Auto re-indexing removes a natural delay window for detection.
Recommended controls:
  • CTRL-RAG-INTEGRITY — Review workflow on KB changes, signed content, anomaly detection on retrieval distributions.

RUN-OUTPUT-INJECTION — outputinjection

MEDIUM
Evidence
LLM output rendered as Markdown in the support agent's UI. No mention of HTML escaping or link validation.
Reasoning
Markdown rendering of attacker-influenced output can result in XSS, phishing links, or malformed UI. Risk depends on Markdown renderer config.
Recommended controls:
  • CTRL-OUTPUT-HANDLING — Escape output, restrict Markdown features, validate links against allowlist.

Testing

TEST-ADVERSARIAL — adversarialsimulation

MEDIUM
Evidence
Architecture document does not mention any red-teaming, adversarial simulation, or jailbreak testing process.
Reasoning
No evidence the system has been adversarially tested. For a public-facing LLM application this is a material gap.
Recommended controls:
  • CTRL-RED-TEAM — Establish adversarial testing cadence: jailbreak corpus, automated probes, human red team before major releases.

Privacy

PRIV-MINIMIZATION — dataminimisation

LOW
Evidence
User messages are logged with full content for 30 days. PII not redacted before logging.
Reasoning
Necessary for debugging but excessive retention period and no minimisation. Hygiene issue.
Recommended controls:
  • CTRL-DATA-MIN — Shorten retention, redact PII before logging, document lawful basis.
