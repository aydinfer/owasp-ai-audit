# Regression: vercel/ai-chatbot, v0.2.2 → v1.0.0

The empirical signal that the v1.0.0 completeness contract holds. Same target
(`vercel/ai-chatbot`), same skill, two contracts. Artifacts in this directory:
[`findings-v0.2.2.json`](vercel-ai-chatbot/findings-v0.2.2.json) (baseline) and
[`findings.json`](vercel-ai-chatbot/findings.json) +
[`dashboard.html`](vercel-ai-chatbot/dashboard.html) (v1.0.0).

## Headline

| | **v0.2.2** | **v1.0.0** |
|---|---|---|
| Taxonomy entries with an explicit verdict | **26 / 97** (~27%) | **97 / 97** — every applicable entry (85/85) + 12 justified predictive N/A |
| Substantive verdicts (non-N/A) | ~13 | **49** (26 MEDIUM + 23 LOW) |
| Finding cards | 26 | 20 |
| Highest severity | **HIGH ×1** | **MEDIUM** (capped) |
| Coverage measured? | **no** | **yes — 8 layers, all green (mean 100%)** |

| Evidence class on findings | none | **all 20 `static`** |
| **Reported posture** | **Acceptable** | **Concerning** |

**Regression signal: MORE verdicts (97 vs 26) AND a LOWER posture (Concerning < Acceptable).** Both halves of the contract fire.

## What the v0.2.2 run actually was

A highlights reel that *looked* thorough. It wrote 26 finding cards — but on a 97-entry taxonomy that is 26 entries touched and **71 never adjudicated**, with no record of which were skipped or why. It reported **"Acceptable"** with:

- **no coverage measurement** — nothing said it had examined ~27% of the taxonomy;
- **a single `HIGH`** on `INPUT-AIRESOURCEEXHAUSTION` carrying **no probe and no evidence class** — and, as the v1.0.0 read of `lib/ratelimit.ts` + `entitlements.ts` shows, the app *does* rate-limit (atomic Redis INCR + per-user hourly entitlements), so that HIGH was not just unprobed but most likely wrong;
- **a rollup label that didn't even follow `verdict-rules.md`** — its own categories were AMBER, which is "Concerning," not "Acceptable."

Three independent ways to overstate confidence, in one report.

## What v1.0.0 makes impossible

1. **Coverage is structural and caps the posture.** Eight layers, each with a formula in [`SKILL.md`](../SKILL.md); the cap is computed in [`scripts/lib/coverage.js`](../scripts/lib/coverage.js) from the numerators you record — a hand-edited `percent` cannot move it. Had any layer dropped below 90%, the page-one banner would read **"Partial …"** or **"Screen only — not an audit"** regardless of how clean the cards looked. Here the audit genuinely earned ≥90% on all eight (L1 27/27 files read, L2 85/85 applicable verdicted, L3 30/30 cells, L4 7/7, L5 0/0 vacuous, L6 7/7 obligations, L7 4/4, L8 4/4), so the *graded* posture stands — but the graded posture is now the honest one.

2. **The honest grade is worse, and you can't relabel it.** With every applicable entry adjudicated, three categories are AMBER (general-controls, input-threats, runtime) → `verdict-rules.md` says **Concerning**. v1.0.0 sets the rollup deterministically from the complete verdict ledger; the "Acceptable" gloss is no longer reachable.

3. **The unprobed HIGH can't exist.** L5 caps severity by evidence class: `static` → MEDIUM, `reasoned-probe` → HIGH, `demonstrated` → CRITICAL. This audit ran no live probes, so every one of its 20 findings is `static` and **capped at MEDIUM** — and `scripts/lib/coverage.js` flags any HIGH+/no-class finding as a violation (0 here). The v0.2.2 HIGH would be a flagged violation today.

## The completeness credential

The v1.0.0 dashboard leads with a **coverage panel before any findings** and closes with a **verdict-ledger appendix** — 97 rows, every applicable entry with a verdict and rationale, all 48 N/A shaded and justified by surface absence (no `rag-embeddings` → augmentation threats N/A; no `training` → poisoning-of-own-model N/A; no predictive classifier → all 12 evasion entries N/A). That ledger is the thing v0.2.2 could not produce: proof of what was looked at, not a selection of what was found.

## Reproduce

```bash
node scripts/enumerate-ai-surfaces.js /path/to/vercel-ai-chatbot --out surfaces.json
# run the SKILL.md eight-layer workflow in Claude Code against the repo
node scripts/render-dashboard.js benchmarks/vercel-ai-chatbot/findings.json out.html
```

Top v1.0.0 finding themes (all MEDIUM, `static`): no system/user prompt segregation (root of direct/indirect/agentic injection); model output flowing into the Pyodide client exec + markdown render without encoding; precise geolocation injected into every prompt with indefinite retention (GDPR/CPRA); no prompt-injection testing/monitoring/output moderation; unpinned model + CDN supply chain; document `DELETE` dereferencing `document.userId` before a not-found guard the `GET` path has.
