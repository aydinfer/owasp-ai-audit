# Regression: vercel/ai-chatbot, v0.2.2 → v1.0.0

The empirical signal that the v1.0.0 completeness contract holds. Same target
(`vercel/ai-chatbot`), same skill, two contracts. Artifacts in this directory:
[`findings-v0.2.2.json`](vercel-ai-chatbot/findings-v0.2.2.json) (baseline) and
[`findings.json`](vercel-ai-chatbot/findings.json) +
[`dashboard.html`](vercel-ai-chatbot/dashboard.html) (v1.0.0).

**Regression signal: MORE verdicts (97 vs 26) AND a LOWER posture (Concerning < Acceptable).** Both halves of the contract fire — and the two HIGHs are *earned* (probed), not asserted.

## What the v0.2.2 run actually was

A highlights reel that *looked* thorough. It wrote 26 finding cards — but on a 97-entry taxonomy that is 26 entries touched and **71 never adjudicated**, with no record of which were skipped or why. It reported **"Acceptable"** with:

- **no coverage measurement** — nothing said it had examined ~27% of the taxonomy;
- **a single `HIGH`** on `INPUT-AIRESOURCEEXHAUSTION` carrying **no probe and no evidence class**, and reasoned from the wrong mechanism — it claimed a fail-open / guest-bypassable rate limiter, but the limiter isn't fail-open. (The entry *does* deserve HIGH — the v1.0.0 probe finds the real mechanism below — but v0.2.2 had no evidence standing to assert it; it was right by accident.);
- **a rollup label that didn't even follow `verdict-rules.md`** — its own categories were AMBER, which is "Concerning," not "Acceptable."

Three independent ways to overstate confidence, in one report.

## What v1.0.0 makes impossible

1. **Coverage is structural and caps the posture.** Eight layers, each with a formula in [`SKILL.md`](../SKILL.md); the cap is computed in [`scripts/lib/coverage.js`](../scripts/lib/coverage.js) from the numerators you record — a hand-edited `percent` cannot move it. Had any layer dropped below 90%, the page-one banner would read **"Partial …"** or **"Screen only — not an audit"** regardless of how clean the cards looked. Here the audit genuinely earned ≥90% on all eight (L1 27/27 files read, L2 85/85 applicable verdicted, L3 30/30 cells, L4 7/7, L5 2/2 HIGH findings probed, L6 7/7 obligations, L7 4/4, L8 4/4), so the *graded* posture stands — but the graded posture is now the honest one.

2. **The honest grade is worse, and you can't relabel it.** With every applicable entry adjudicated, four categories are AMBER (general-controls, input-threats, runtime, privacy) → `verdict-rules.md` says **Concerning**. v1.0.0 sets the rollup deterministically from the complete verdict ledger; the "Acceptable" gloss is no longer reachable.

3. **Severity has to be earned.** L5 caps severity by evidence class: `static` → MEDIUM, `reasoned-probe` → HIGH, `demonstrated` → CRITICAL. A deep reasoned-probe pass ran the top attack paths — and most locks **held**: cross-tenant IDOR is re-checked on every route, the document tools re-derive authority from the session (so a prompt-injected `updateDocument({id:<foreign>})` returns `Forbidden`), model/entitlement selection is allowlisted, and the Pyodide artifact runs in the user's *own* browser behind an explicit Run click. Two findings earned **HIGH** with a verbatim probe and a named depended-on behavior — `INPUT-AIRESOURCEEXHAUSTION` (the `role:'user'` rate counter leaves tool-spawned nested `streamText` calls uncounted → ~5× free fan-out per counted message) and `PRIV-AIPRIVACY` (precise geolocation injected into every system prompt, read back on demand, retained indefinitely). **Zero** reached CRITICAL/`demonstrated`, because the app was not run. And `scripts/lib/coverage.js` flags any HIGH+/no-class finding as a violation (0 here) — so the v0.2.2 unprobed HIGH would be rejected today.

## The completeness credential

The v1.0.0 dashboard leads with a **coverage panel before any findings** and closes with a **verdict-ledger appendix** — 97 rows, every applicable entry with a verdict and rationale, all 48 N/A shaded and justified by surface absence (no `rag-embeddings` → augmentation threats N/A; no `training` → poisoning-of-own-model N/A; no predictive classifier → all 12 evasion entries N/A). That ledger is the thing v0.2.2 could not produce: proof of what was looked at, not a selection of what was found.

## Reproduce

```bash
node scripts/enumerate-ai-surfaces.js /path/to/vercel-ai-chatbot --out surfaces.json
# run the SKILL.md eight-layer workflow in Claude Code against the repo
node scripts/render-dashboard.js benchmarks/vercel-ai-chatbot/findings.json out.html
```

v1.0.0 findings, by evidence class:

- **HIGH (`reasoned-probe`):** cost amplification via uncounted nested `streamText` under `stepCountIs(5)` (`INPUT-AIRESOURCEEXHAUSTION`); always-on precise-geolocation injection + indefinite retention (`PRIV-AIPRIVACY`).
- **Probed but the lock held (graded MEDIUM/LOW honestly):** direct/indirect prompt injection into tools (tools re-check ownership), agentic tool abuse, Pyodide self-XSS blast radius, the `DELETE` not-found `undefined.userId` deref (self-inflicted 500, not cross-tenant).
- **`static` MEDIUM (14):** no system/user prompt segregation; model output into Pyodide exec + markdown render without encoding; no PI testing/monitoring/output moderation; unpinned model + CDN supply chain; retention/transparency/governance gaps.
